## 1. プロジェクト概要

- **名称**: Thanks Card（鮑屋サンクスカード Web アプリ）
- **目的**: 鮑屋グループの紙ベースサンクスカードを Web 化し、行動特性評価との連動余地を確保しつつ、感謝の文化を可視化・活性化する
- **クライアント**: AWB（株式会社鮑屋・株式会社AWABIYA HD）
- **法務契約**: アイアルマーズ本体・飯澤=コムフォワード・**角田=天神経営**

### 4.3 確定済 仕様（やる・やらない）

#### ✅ 採用

- 累計表記は「2025年4月～累計」起点（マイページ・統計）
- メダル風バッジUI（金/銀/銅グラデ + 順位バッジ）
- バッジロジック新仕様:
  - **月 × 全体集計**で 3指標（hearts / received / sent）
  - **最低3件閾値**（`BADGE_MIN_THRESHOLD = 3`）
  - **同点全員ランクイン**（ユニーク値ベース上位3枠）
- 「グッドサンクス」呼称統一（旧「ハート投票」を改名）
- 好事例タブ（送受両側集計）
- ランキング: もらった・贈った・グッドサンクス・好事例の4タブ
- 全部見るのカードはランダム表示（再クリックで再シャッフル）
- 名前の前に小さく所属を併記表示

#### ❌ 廃止 / 不採用

- **カテゴリ機能**（行動特性5項目との紐付け）
  - 廃止理由: MVV 未確定のため一旦撤去。MVV 確定後に再設計の余地は残す
  - DB: `categories` / `card_categories` テーブル drop migration 用意済（`supabase/migrations/2026-04-27_drop_categories.sql`・本番未適用）
- **拠点別集計セクション**（全部見る画面）
  - 廃止理由: 集計の意味付けが固まらず、再設計まで一旦非表示
- **拠点タグ表示**（カードのメタ情報として）
  - → 名前前の所属併記に置き換え
- **バースデーピッカー認証**（旧路線）
  - → 氏名ログインに置き換え
- **Microsoft Forms 路線**（路線A）
  - → Next.js アプリ路線（路線B）に統合済
